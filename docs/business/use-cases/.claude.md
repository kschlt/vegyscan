# Use Cases - AI Instructions

## Purpose

Each use case describes ONE specific interaction between a user and the system.

## Creating New Use Cases

### When to create
- New feature identified in business analysis
- Existing use case becomes too complex (split it)

### Process
1. Copy `_template.md` to new file
2. Name it: `UC-XXX-short-description.md` (XXX = next number, zero-padded)
3. Fill all sections
4. Link to relevant personas and requirements
5. Update `README.md` with new UC in list

### Naming Rules
- ✅ `UC-001-scan-ingredient-list.md`
- ✅ `UC-012-save-favorite-products.md`
- ❌ `use-case-1.md` (no UC prefix)
- ❌ `UC-1-scan.md` (not zero-padded)
- ❌ `scan-ingredients.md` (no UC number)

## Structure Requirements

Each use case MUST have:
- **Title**: Clear, action-oriented
- **ID**: UC-XXX
- **Actors**: Link to `../personas.md`
- **Preconditions**: What must be true before
- **Main Flow**: Step-by-step happy path
- **Alternative Flows**: Variations
- **Exception Flows**: Error cases
- **Postconditions**: System state after success

## Content Rules

**Include:**
- ✅ User actions and system responses
- ✅ Links to personas: `[Health-conscious consumer](../personas.md#persona-name)`
- ✅ Links to requirements: `[REQ-F-001](../requirements/functional.md#req-f-001)`
- ✅ Business logic and rules

**Exclude:**
- ❌ Implementation details (how to code it)
- ❌ UI mockups (use requirements/non-functional.md for UX requirements)
- ❌ Technology choices (→ architecture/adr/)

## Updating Existing Use Cases

### Allowed updates
- ✅ Add alternative/exception flows
- ✅ Clarify existing steps
- ✅ Add cross-references
- ✅ Update status

### Requires careful review
- ⚠️ Changing main flow significantly
- ⚠️ Changing preconditions/postconditions
- ⚠️ Removing flows

**If a use case changes fundamentally:**
1. Consider if it should be a NEW use case
2. Check if linked requirements need updates
3. Verify personas are still accurate

## Cross-References

- Personas: `../personas.md#persona-id`

## Status Values

- **Draft**: Work in progress
- **Review**: Ready for stakeholder review
- **Approved**: Accepted by stakeholders
- **Implemented**: Feature is live
- **Deprecated**: No longer relevant
